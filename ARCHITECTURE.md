## Key Design Decisions

1. **Keyword Extraction First**: Extract meaningful words from input before searching
2. **Batch Searching**: Query API once per keyword, not one big search
3. **Deduplication**: Remove duplicate tokens before scoring
4. **Multi-Factor Scoring**: Weight multiple criteria (keywords, category, market cap, liquidity)
5. **Smart Sorting**: Primary sort by relevance, secondary by market cap
6. **Top 20 Results**: Balance between completeness and usability
7. **Real-Time Data**: 100% from live APIs, no caching
8. **Comprehensive Logging**: Every step logged for debugging

---

**Total lines of code**: ~600 lines of vanilla JavaScript
**External dependencies**: None (pure JS ES6+)
**API calls per search**: 4-6 concurrent requests
**Data freshness**: Real-time from DexScreener
**Browser support**: Chrome, Firefox, Safari, Edge (modern versions)
